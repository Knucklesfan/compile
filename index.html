GNU Compiler Collection

Unsurprisingly, compiling C on Linux is by far the easiest method. No external applications are necessary, in some cases, you might already have the right tools installed! If gcc -v returns an error, however, please continue to these next steps.

Installing GCC

To install GCC, please pick your distro from below.

To install GCC and all other required C resources, please run the following commands.

sudo apt update

followed by

sudo apt install build-essential

To install GCC and all other required C resources, please run the following commands.

sudo pacman -S base-devel

To install GCC and all other required C resources, please run the following commands.

sudo dnf install @development-tools

Compiling and running code

Okay, hope you haven't closed that command prompt! Now you can begin to compile your code. To compile your main.c file, run the following command

gcc main.c

Upon compilation completion, you can now run the exported file a.out like so.

./a.out

You can also run the compilation to export the object with a particular file name.

gcc main.c -o main

To run this code, you only need to change the execution statement.

./main

Installing XCode

There's a couple means of installing XCode on macOS. Please pick your prefered method. I highly recommend installing from the app store.

First, find XCode in the app store, and install it. XCode is available on most Macs running at least MacOS 10.6

Alternatively, for Macs that cannot access the App Store or are too outdated, you can optionally download XCode from Apple's website. You should pick XCode 13. XCode 14 is still not released, and therefore not suitable for this tutorial. (Updated 9/8/2022)

You will need to log in to your Apple ID to access as well.

Download

Creating a new project in XCode

After installation, launching XCode should result in this screen. Click the Create a new XCode project option.

After creating a new project, you should be presented with a new screen. In this screen, you should pick macOS as the platform and Command Line Application as the type.

After selecting a project type, you will be presented with a setup screen. In this screen, you should pick C or C++ as the language. Project names can have any type of spelling, including spaces and forbidden characters. The Team, Organization Identifier, and Bundle Identifier options do not matter, and only apply to App Store application and other distribution necessities.
